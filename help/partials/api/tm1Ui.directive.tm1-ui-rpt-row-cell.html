tm1-ui-rpt-row-cell
directive in module tm1Ui

Description

Use this to retrieve the cell object that was retrieved, for performance consideration. This directive is used in the tbody section of your table element with the active form. Use as part of tm1-ui-rpt-view.

Usage

as element:
<tm1-ui-rpt-row-cell
       tm1-row-data="{object}"
       tm1-column-no="{string}"
       tm1-column-elements="{string}"
       [tm1-picklist-disable-search="{boolean}"]
       [tm1-ui-class="{string}"]>
</tm1-ui-rpt-row-cell>

Directive info

  • This directive creates new scope.

Parameters

ParamTypeDetails
tm1-row-dataobject

The row property of the object defined on the parent tm1-ui-rpt-view. REQUIRED

tm1-column-nostring

A reference to the column number defining the cell on the current row, as per result of the active form query. WATCHED

tm1-column-elementsstring

A reference to the elements in the column defining the cell on the current row, as per result of the active form query. WATCHED

tm1-picklist-disable-search
(optional)
boolean

For enabling/disabling the search text functionality on Cells with Picklist.

(default: "false")

tm1-ui-class
(optional)
string

Add other custom classes into this directive.WATCHED

(default: "tm1-ui-rpt-row-cell")
