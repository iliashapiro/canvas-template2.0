tm1-ui-subnm
directive in module tm1Ui

Description

Allows you to select a specific element on a targeted dimension.

Directive Level APIs

refreshData()
For refreshing the list of elements
getElements()
For retrieving the elements on the current state of this SUBNM

Usage

as element:
<tm1-ui-subnm
       tm1-instance="{string}"
       tm1-dimension="{string}"
       [tm1-subset="{string}"]
       [tm1-attribute="{string}"]
       [tm1-default-element="{string}"]
       [tm1-mdx="{string}"]
       [tm1-show-hierarchy="{boolean}"]
       [tm1-select-only="{boolean}"]
       [tm1-radio-only="{vertical|horizontal}"]
       [tm1-server-mode="{boolean}"]
       [tm1-batch-size="{string}"]
       [tm1-api="{object}"]
       [tm1-on-load="{function}"]
       [tm1-on-change="{function}"]
       [tm1-change="{function}"]
       [tm1-ui-class="{string}"]
       ng-model="{object}">
</tm1-ui-subnm>

tm1-server-mode
(optional)
boolean

Use this to minimize the number of elements for each query. Note that once this is provided, this directive will automatically be in a non-hierarchy search mode.

This is helpful, when querying a dimension with a large number of elements.

Also take note that this is not applicable if tm1-mdx is used. If MDX is needed, then create a Subset with that MDX, and use tm1-subset instead.

Also, due to the nature of the search and in relation to OData compliance as per IBM, ensure that the Alias set in the Subset or the tm1-attribute provided to this directive during tm1-server-mode operation only has alpha numeric characters. So instead of having 'Code & Description' or 'Code and Description', change or use another attribute with a name like: 'CodeAndDescription'.

(default: "false")
