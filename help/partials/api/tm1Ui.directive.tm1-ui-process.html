tm1-ui-process
directive in module tm1Ui

Description

A directive that allows you to call a TI process.

Usage

as element:
<tm1-ui-process
       tm1-instance="{string}"
       tm1-name="{string}"
       [tm1-parameter-*="{string}"]
       [tm1-display-name="{string}"]
       [tm1-message-running="{string}"]
       [tm1-message-success="{string}"]
       [tm1-message-error="{string}"]
       [tm1-ui-class="{string}"]
       [tm1-ui-button-class="{string}"]
       [tm1-after-run="{function}"]
       [tm1-show-dialog="{boolean}"]
       [tm1-refresh-page="{boolean}"]
       [tm1-disabled="{boolean}"]>
</tm1-ui-process>

Directive info

  • This directive creates new scope.

Parameters

ParamTypeDetails
tm1-instancestring

Instance name for this directive to connect to. REQUIRED

tm1-namestring

Name of the ProcessREQUIRED

tm1-parameter-*
(optional)
string

Parameters of the process where the asterisk corresponds to the name of the TI parameter. For example, if the name of a TI parameter is pCostCentre, then this attribute will be tm1-parameter-pCostCentre="1001". WATCHED

tm1-display-name
(optional)
string

Display name of the Process buttonWATCHED

tm1-message-running
(optional)
string

Process running display messageWATCHED

tm1-message-success
(optional)
string

On success display messageWATCHED

tm1-message-error
(optional)
string

On error display messageWATCHED

tm1-ui-class
(optional)
string

Add other custom classes into this directive.WATCHED

(default: "tm1-ui-process")

tm1-ui-button-class
(optional)
string

Add other custom classes to change the appearance of the button. Note that you can alternatively customize the CSS classes: tm1-ui-process-button and tm1-ui-process-cancel-button WATCHED

(default: "tm1-ui-process-button")

tm1-after-run
(optional)
function

A function that will be called after the process has ran. Add a parameter named 'success' into the function for the status of the process just ran.

For example, if you have declared a function on your page's controller, named and with signature: 

	$scope.runOtherThings = function(success){ 
		console.log('Success: %o', success); 

		// do other actions
	};
Call them like:

<tm1-ui-process
		tm1-instance="dev"
		tm1-name="Bedrock.Server.SaveDataAll"
		tm1-after-run="runOtherThings(success)"
>
</tm1-ui-process>
