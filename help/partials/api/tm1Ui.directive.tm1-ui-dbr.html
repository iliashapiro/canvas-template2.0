tm1-ui-dbr
directive in module tm1Ui

Description

A DBR-like implementation for retrieving cube data from TM1.

For alternate hierarchy, consider a Product dimension used in a cube having a hierarchy by attribute named Location. To get the values of Bikes product, on the AU market:

tm1-elements=".., Bikes && Location::AU, .."

Directive Level APIs

onUpdate(status, data)
A function called after an update request has been initiated.
setContextTimeout(seconds)
A function, that sets how long will the context button be visible on activation. Default is 5 seconds.

tm1-before-update
(optional)
function

A function that will be called when an update event executes. The signature of the function that you can should have '$event' as one of the parameter. The next step would just be to call the cancel() function of the $event object received.

For example, if you have a function named checkData, it should be written as 'tm1-before-update="checkData($event)"'. To cancel the update after your validation

$scope.checkData = function($event){
		// do some validation here
		if($scope.dataIsInvalid){
			$event.cancel();
		}
};

Note: If you want to return the response to DBR to prevent the update, use SYNCHRONOUS mode (i.g. use jQuery's ajax() instead).

You can also pass you own custom message validation by calling the cancel function with the message as parameter. It should look like below:

$scope.checkData = function($event){
		// do some validation here
		if($scope.dataIsInvalid){
			$event.cancel('Invalid Value has been detected!');
		}
};
