tm1-ui-login
directive in module tm1Ui

Description

A component that displays a login prompt, for connecting into TM1 instances as identified in instances.json file.

Usage

as element:
<tm1-ui-login
       [tm1-after-login="{function}"]
       [tm1-default-instance="{string}"]
       [tm1-ui-class="{string}"]>
</tm1-ui-login>

Directive info

  • This directive creates new scope.

Parameters

ParamTypeDetails
tm1-after-login
(optional)
function

A function that will be called after a successful login. The signature of the function that you can add should only have 'instance' as the parameter.

For example, if you have a function named checkData, it should be written as 'tm1-after-login="checkData(instance)"'. Take note too that utilizing this attribute means you will need to handle what happens after the user has logged in.

tm1-default-instance
(optional)
string

The default instance to set when showing the login prompt.

tm1-ui-class
(optional)
string

Add other custom classes into this directive.WATCHED

(default: "tm1-ui-login")
