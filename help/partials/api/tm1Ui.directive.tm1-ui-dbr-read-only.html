tm1-ui-dbr-read-only
directive in module tm1Ui

Description

A DBR-like implementation for retrieving cube data from TM1 that is always read-only and to be used with ng-model to retrieve numeric values

For alternate hierarchy, consider a Product dimension used in a cube having a hierarchy by attribute named Location. To get the values of Bikes product, on the AU market:

tm1-elements=".., Bikes && Location::AU, .."

Usage

as element:
<tm1-ui-dbr-read-only
       tm1-instance="{string}"
       tm1-cube="{string}"
       tm1-elements="{string}"
       [tm1-comment-cube="{string}"]
       [tm1-comment-elements="{string}"]
       [tm1-comment-format-html-buttons="{Array}"]
       [tm1-comment-read-only="{boolean}"]
       [tm1-refresh-group="{string}"]
       [tm1-sandbox="{string}"]
       [tm1-sandbox-class="{string}"]
       [tm1-hide-reference="{boolean}"]
       [tm1-hide-comment="{boolean}"]
       [tm1-hide-spread="{boolean}"]
       [tm1-hide-drill="{boolean}"]
       [tm1-hide-trace-calculation="{boolean}"]
       [tm1-hide-context-menu="{boolean}"]
       [tm1-add-context-menu="{Array}"]
       [tm1-format-percentage="{boolean}"]
       [tm1-format-parenthesis="{boolean}"]
       [tm1-data-decimal="{string}"]
       [tm1-data-scale="{string}"]
       [tm1-ui-class="{string}"]
       [tm1-accounting-settings-override="{object}"]
       [tm1-options="{object}"]
       [tm1-drill-model="{object}"]
       [ng-model="{object}"]>
</tm1-ui-dbr-read-only>

Add menus that can be called from the context button. The function will have cell properties available.

As an example, to call a function called myContextMenuAction, setup as follows:

On your JS file,

$scope.myContextMenus = [

{

display: 'Click Me!',

action: function(data){

console.debug('myContextMenuAction() %o', data);

}

}

];

Then on your HTML,

<tm1-ui-dbr ... tm1-add-context-menu="myContextMenus"><tm1-ui-dbr>
