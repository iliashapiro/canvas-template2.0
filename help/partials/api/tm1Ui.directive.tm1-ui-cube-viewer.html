tm1-ui-cube-viewer
directive in module tm1Ui

Description

This directive helps you to slice and dice a cube, and also work with views and subsets.

Dimension Configuration object properties (for use with tm1-options.rows|columns|titles)

  • dimension - Dimension name
  • hierarchy - Dimension hierarchy
  • mdx - MDX statement that will be used if the Subset does not exist. This will be used to create the Subset.
  • subset - Dimension subset to be used for the view. Preference will be given to this over the MDX. If not existing, this will be created, using the MDX on this configuration.
  • alias - Dimension alias to be used for the dimension elements of this view. This will override the default defined on the subset for display only (will not override TM1).
  • elements - A comma-separated list of element(s) or aliases to be used on the dimension of this view. This will override Subset and/or MDX if set. (will not override TM1).
  • elementSelected - Dimension element to used for specifying an element to be selected on a dimension. Applicable for hierarchies in the TITLE axis only.

Directive Level APIs

refreshData()
For refreshing the list of data on this directive
getTableData()
For retrieving the data object of this directive which functions that allow you to control the pagination for example.
isBuiltinPaginationVisible()
To get the status of whether the pagination controls is visible or not.
toggleBuiltinPagination()
For toggling the visibility of the pagination controls.

Usage

as element:
<tm1-ui-cube-viewer
       tm1-instance="{string}"
       tm1-cube="{string}"
       [tm1-view="{string}"]
       [tm1-options="{object}"]
       [tm1-ui-class="{string}"]
       [tm1-api="{object}"]>
</tm1-ui-cube-viewer>

Directive info

  • This directive creates new scope.
