tm1-ui-upload
directive in module tm1Ui

Description

A component for uploading files into your Canvas application, by instance.

On after a successful upload, information regarding the file can be received via the tm1AfterUpload attribute. Assign a function with a signature of function(file).

For example:

	<tm1-ui-upload tm1-instance="dev" tm1-after-upload="myFunction(file)"></tm1-ui-upload>

Usage

as element:
<tm1-ui-upload
       tm1-instance="{string}"
       [tm1-max-size="{string}"]
       [tm1-display-name="{string}"]
       [tm1-image-options="{string}"]
       [tm1-path="{string}"]
       [tm1-retain-name="{boolean}"]
       [tm1-after-upload="{function}"]
       [tm1-ui-class="{string}"]>
</tm1-ui-upload>

Directive info

  • This directive creates new scope.

Parameters

ParamTypeDetails
tm1-instancestring

Instance name for this directive to connect to. REQUIRED

tm1-max-size
(optional)
string

Size in MB of the maximum file size that can be uploaded.

(default: "5")

tm1-display-name
(optional)
string

Button display name for prompting user for file selection.

(default: "Select File")

tm1-image-options
(optional)
string

Options that can be passed on to affect the upload of images. See ngf-resize property here for full reference.

tm1-path
(optional)
string

Additional path to be appended in relation to where the file will be saved.WATCHED

tm1-retain-name
(optional)
boolean

If set to true, this will keep the name of the file when uploaded. Note that this will overwrite an existing file.WATCHED

(default: false)

tm1-after-upload
(optional)
function

A function that will be called after a successful upload. The signature of the function that you can add should have 'file' as one of the parameter(s).

For example, if you have a function named checkData, it should be written as 'tm1-after-upload="checkData(file)"'.

tm1-ui-class
(optional)
string

Add other custom classes into this directive.WATCHED

(default: "tm1-ui-upload")
